## Defects and fix

1. Feature Details used visually adjacent `<label>` elements without `for`/`id` associations. The same omission affected inline table inputs and dropdowns. The fix assigns stable per-feature control IDs, associates every visible label, and gives table editors a contextual name such as `Name, feature 1`.
2. Print/PDF preparation counted only `layer.parsed.rows`, while active projects store drawn features in `features` and `parsed.records`. The fix normalizes the supported representations and uses the maximum representation count per layer to avoid both zero counts and double counting.
